> AI 说到一半闭嘴了，日志干干净净。审批模式设成 auto，一个编码问题差点清了系统根目录。用 Hermes 踩过的坑和攒下的玩法，整理成一份配置速查 + 七个实用技巧。

## TL;DR——配置速查

先把这几项设了：

```
# ~/.hermes/config.yaml

model:
  max_tokens: 131072          # 不设就截断，GLM-5.1 直接拉满
  context_length: 200000      # 显式写，不然压缩阈值算不准

approvals:
  mode: smart                 # manual 太烦，auto 太险，smart 用 AI 管 AI

checkpoints:
  enabled: true               # 默认关闭，打开才有快照回滚
  max_snapshots: 50
```

`/sethome` 设好定时任务的收件地址——微信端直接在聊天窗口输这个命令就行。

**🤖 Agent 一键配置**：直接把下面这段发给你的 AI agent，让它自己读配置、改配置、验证：

```
请帮我配置 Hermes Agent 的关键参数。

1. 读取 ~/.hermes/config.yaml，确认当前使用的模型（model 段）
2. 查询该模型的上下文窗口长度和最大输出 tokens（查模型官方文档或 models.dev）
3. 根据查到的参数，确保以下配置项存在且值正确（已有的字段改值，缺失的字段补充）：
   - model.context_length: <该模型的上下文窗口长度>（不写压缩阈值算不准）
   - model.max_tokens: <该模型的最大输出 tokens>（不设会被截断）
   - approvals.mode: smart（manual 太烦 auto 太险，smart 让 AI 帮你管审批）
   - checkpoints.enabled: true（默认关闭，不开就没法回滚）
   - checkpoints.max_snapshots: 50
4. 改完后重新读取文件，逐项确认修改是否生效
5. 提醒我：修改配置后需要重启网关（/restart）才能生效，并简要说明每项改动的作用
```

注意：checkpoints 目前只在 CLI（本机终端）生效，Telegram/微信端文件操作不产生快照。

## 一、AI 说话说半截——沉默失败比报错更危险

有一天我让小虾子改一段配置，她说了一大段解释，说到一半突然停了。话没说完，后面没了。

因为原来用的都比较正常，那么突然在特定的情况下它就停止了回复，而且你在追问的时候它又会正常回复。并且没有报错，那就证明它不是这种程序的问题，更可能是设置问题。

所以我说明问题让她去自查：Hermes 的 `config.yaml` 里配了模型，但**没有设 `max_tokens`**。API 在不指定这个参数时，默认输出长度只有 4096 tokens。一个中文字大约 1-2 个 token，4096 tokens 实际只能输出 2000-3000 中文字。稍微长一点的回复就超了，API 返回 `finish_reason=length`，但 Hermes 的 agent loop 把它当正常结束处理了。

**没有任何报错。日志干干净净。** API 调用成功了，只是"成功了一半"。

修复很简单，把 `max_tokens` 设成模型支持的上限：

```
model:
  max_tokens: 131072    # GLM-5.1 最大输出 128K tokens
```

设得比模型上限高没关系，API 会自动按模型天花板截断。但设得低了，模型真的会说到一半突然就没有了。

**沉默失败比报错更危险。** 报错你会查，截断你可能以为"模型就这样"，莫名其妙。

## 二、审批模式——从全堵到全放，中间有条路

Hermes 有三种审批模式：manual、smart、auto。

manual 是默认的，每个可疑命令都问你。刚开始用或者在生产环境，这个模式合适，但用熟了之后，你会被审批弹窗烦死——改个配置文件要确认，装个依赖要确认，创建个目录也要确认。

auto 是全部放行。你对环境很了解、运行也很稳定了，直接用 auto 确实省事。但风险不可预估。上周我就差点翻车——不是 AI 故意搞事，是代码解析时有个编码问题没正确解析引号，一个正常的文件操作路径变成了系统根目录！如果不是系统文件占用，整个盘就没了。

![](/assets/img/news/用 Hermes 两天，差点把我系统送走：7个踩坑玩儿法实践-1.png)

**不好意思我搞混了，这个是 Claude Code 干的，先不改了 - -#**

**smart 模式是被忽视的第三条路。** 系统会用另一个模型当安全员，帮你判断命令风险级别——低风险的自动放行，高风险的才问你。买菜自动过，大额转账才打确认电话。代价就是多一次 API 调用。如果你用的是 OpenRouter 这类聚合平台，它会自动选便宜的模型来审；如果你只配了一个模型，就用那个模型自己审自己。

```
approvals:
  mode: smart
  timeout: 60
```

日常使用推荐 smart。刚入门用 manual，环境熟了再切 smart。auto 慎用。

## 三、Checkpoints——改错了？回到过去

Hermes 有个 checkpoint 检查点系统。每次文件修改前都会自动创建快照。回滚的时候不只是恢复文件，还会撤销对应的对话轮次——AI 不会"记得"已经被撤销的改动，它真的回到那个时间点了。

而且它用的是独立的 shadow git 仓库来管理快照，不会碰你项目自己的 `.git`。修改代码这些方面你可以毫无压力地用。

但 checkpoints 默认是关闭的。你需要先把它打开：

```
checkpoints:
  enabled: true
  max_snapshots: 50
```

**开了之后怎么用？** 直接在聊天里输 `/rollback`，会列出所有快照，带编号和时间。觉得改坏了，`/rollback 3` 回到第 3 号快照。不确定要不要回？先 `/rollback diff 3` 看看那次快照之后改了什么。只想恢复一个文件？`/rollback 3 config.yaml` 只动那一个文件，其他改动保留。

**注意一点：Checkpoint 目前只在本机命令行（CLI）里生效。** 通过 Telegram、微信等消息平台让 AI 修改文件时，不会产生快照，`/rollback` 也查不到记录。如果你需要回滚保护，文件修改操作在电脑终端里做。

## 四、reset、new、restart

我自己犯过一次。本来想重启网关，脑子：要输 `/restart`，手：打成了 `/reset`。回车。

对话没了。干干净净。

在微信端这就比较尴尬了——`reset` 之后没有办法通过 `resume` 回到原来的会话。你虽然可以用 `session search` 找到旧对话的内容，但没法无缝接上继续聊。

这三个命令有迷惑性：

*   • **`/new`**：清空当前对话，开始一个全新的会话。旧会话的数据还在，可以通过命令行恢复。
    
*   • **`/reset`**：效果和 `/new` 一样，是个别名。但这个词更容易让人误解为"重置状态回到某个点"。
    
*   • **`/restart`**：重启 Hermes 的网关进程。不清空对话，只是重新连接。会话数据完整保留。
    
*   • **`/resume`**（或命令行 `hermes -c`）：恢复到上一次的会话，在终端里可以完美接续。
    

如果你在微信或 Telegram 上误操作了 `/reset`，别慌。旧会话的数据没丢，只是在 IM 端没法直接接回去。回到电脑终端，用 `hermes sessions list` 找到旧会话的 ID，然后 `hermes --resume <session_id>` 就能完整恢复。

## 五、会话管理——IM 端和命令行是两个世界

![](/assets/img/news/用 Hermes 两天，差点把我系统送走：7个踩坑玩儿法实践-2.jpg)

Hermes 支持你在电脑上用命令行交互，也支持微信、Telegram 这些即时通讯工具。但两者在会话管理上有比较大的区别。

**会话恢复只支持命令行。** 如果你在微信或 Telegram 上中断了一个会话——比如手滑了 `/reset`，或者长时间没聊自动断了——你没法在 IM 端切回原来的 session。能做到的是：在新对话里告诉 AI 去搜索以前的会话内容，它用 `session_search` 工具可以搜到所有历史对话，把上下文信息拉出来，在新窗口继续用。

这比 memory 更深。memory 是随身便签本，每次都带着但容量有限（2200 字符）。session 是完整日记本，无限制，需要时翻阅。你不需要记得哪天聊的、在哪个平台聊的，直接跟 AI 说"之前我们讨论过 xxx"，它会自动去搜。

命令行下的会话管理命令：

```
hermes sessions list              # 列出所有历史会话
hermes sessions browse            # 交互式浏览 + 搜索
hermes --resume <session_id>      # 直接恢复某个会话
hermes -c                         # 恢复最近一个
```

每个重要会话用 `/title` 命名，日后好找。不需要一开始就设置，什么时候觉得这个 session 需要记住、可能要手动恢复的时候再设。搜索的时候 title 其实不重要，但给会话起个名字方便你在列表里快速辨认。

## 六、语音对话——从虚拟到现实的那个瞬间

## 七、/sethome——定时任务的"收件地址"

这个功能看着不起眼，但如果你用 Hermes 做定时任务，就很关键了。

`/sethome` 设置你的主频道——定时任务的执行结果会发到这个频道。如果不设，Hermes 可能会设了任务但不知道往哪儿发结果。你等了半天没收到消息，以为任务没跑，其实跑完了，只是发到了一个你不在的地方。

用 Hermes 做提醒、做定时汇总、做监控类任务的，记得先确认一下 `/sethome`。
