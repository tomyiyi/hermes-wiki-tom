## 核心要点

Hermes Agent 是由 **Nous Research** 团队开发的一款开源自主 AI 智能体，主打"自我进化"能力。与传统 AI 助手"用完即忘"不同，它能随着使用不断成长。

**GitHub 星标：10万+（2026年4月）**

---

## 三层核心架构

### 第一层：持久记忆
- MEMORY.md：项目环境、踩坑记录、核心约定
- USER.md：用户语言偏好、回答风格
- 历史对话存储在本地 SQLite，可随时追溯
- 用户纠正错误时说"记住此点"，系统主动更新

### 第二层：技能系统（创新亮点）
- 每当完成复杂任务，自动将过程固化为 **SKILL.md** 文件
- 包含触发条件、操作步骤、已知问题
- 下次类似任务直接调用，无需重新推理
- "渐进式披露"策略：先浏览技能目录，仅在需要时详细读取

### 第三层：用户建模
- 集成 Honcho 系统，分析用户行为习惯
- 判断用户偏好：直接给代码，还是先讨论方案？
- 随交互次数增加，回应愈发贴合需求

---

## 自学习闭环机制

三层能力通过自学习机制形成闭环：
1. 记忆为上下文提供支持
2. 技能沉淀经验
3. 用户建模优化交互
4. 完成后自动复盘，分析失败步骤，优化流程，生成/改进技能

---

## 工程能力

| 能力 | 说明 |
|------|------|
| 全平台消息接入 | Telegram、Discord、Slack、WhatsApp、Signal、邮件；微信桥接工具可用 |
| 定时任务 | 内置 cron 调度器，自然语言配置（如"每日8点抓取新闻发摘要"） |
| 并行任务 | 主Agent拆解任务派发给多个子Agent并行处理 |
| 部署方式 | 本地/Docker/SSH/无服务器/HPC集群；5美元/月VPS即可运行 |
| 模型兼容 | 支持 OpenAI、Anthropic、DeepSeek、智谱GLM、Kimi、MiniMax 等20+提供商 |
| 上下文要求 | 至少64K tokens，主流模型均满足 |

**官方推荐：** 日常任务用 Kimi，深度任务用 GPT-4o/Claude，隐私场景用本地 Ollama

## 局限与风险

- 技能生态尚在完善，部分功能存在待解决问题
- 复杂推理的稳定性受底层模型限制
- 建议从非核心业务场景开始试点

## 八卦：被曝抄袭

据 [腾讯新闻转载](https://so.html5.qq.com/page/real/search_news?docid=70000021_14969e0607971452)，Hermes Agent 被 EvoMap 团队指控"结构性同构"——几乎每一个核心模块都与该项目雷同：
- 同样的"任务完成后自动提取可复用资产"的闭环范式
- 同样的三层记忆体系（持久事实层 + 程序性记忆层 + 历史搜索层）
- 同样的"技能在使用中自我改进"

Hermes Agent 2025年7月22日立项后有长达半年空档期，2026年1月31日后才重新更新，2026年3月6日品牌定位更改为"自进化AI智能体"。
