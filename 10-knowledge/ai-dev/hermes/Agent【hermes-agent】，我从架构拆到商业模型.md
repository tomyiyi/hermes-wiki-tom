# 两个月，58k stars，382个贡献者，社区里到处是「我从 OpenClaw 迁移过来了」的帖子。

这个叫 Hermes Agent 的开源项目，到底凭什么？

![](/assets/img/news/Agent【hermes-agent】，我从架构拆到商业模型-0.png)

我也好奇，所以直接上手装了一个。

花了2天时间，我把它的架构从上到下拆了个遍。今天这篇文章，就是把拆解过程完整分享出来。

**简单说它是一个部署在你自己服务器上、7×24 在线的 AI Agent。**

跑在一台 5 美元的 VPS 上就行，你可以通过 Telegram、Discord、飞书、钉钉等 15 个平台跟它对话。它有自己的记忆、会自动学新技能、甚至能在你睡觉的时候自己执行定时任务。

如果非要打个比方的话：Claude Code 像一个坐在你工位旁的实习生，你在的时候帮你干活，你走了他也下班了。Hermes Agent 更像一个你雇的远程全职员工，有自己的工位、自己的电脑，你不在他也在干活。

为了帮大家建立全局认知，我画了一张 Hermes Agent 的架构全景图：

![](/assets/img/news/Agent【hermes-agent】，我从架构拆到商业模型-1.png)

五层结构从上到下：接入层（Gateway）负责连接 15+ 平台，Agent Core 是大脑（负责思考和执行循环），Memory + Skills + Tools 是能力基底，Terminal Backends 是执行环境（从本地到 Serverless），最底下是数据飞轮（用户轨迹反哺模型训练）。

后面几个章节，我会逐层拆开来讲。

从表里可以看出，Claude Code 跟后两者根本不在同一个赛道。真正的对手是 OpenClaw 和 Hermes Agent，它们都在做「持久化个人 Agent」这件事，但设计哲学有一个根本分歧：OpenClaw 的技能靠人写，Hermes 的技能靠 Agent 自己从经验中学。

## 四层记忆架构：它怎么做到永远不忘

你用 ChatGPT 也好，用 Claude 也好，有没有遇到过这种情况：上周刚聊完一个方案，这周再问它，完全不记得了。每次新对话都像面对一个刚入职的实习生，所有上下文从头交代一遍。

Hermes Agent 说它解决了这个问题。我拆完它的记忆系统后发现，它不是简单加了一个记忆功能，而是把记忆拆成了四个层级，每一层解决不同时间尺度的遗忘。

> ![](/assets/img/news/Agent【hermes-agent】，我从架构拆到商业模型-2.png)

**第一层：会话内上下文**

这个最基础，就是当前对话的上下文窗口，所有 LLM 产品都有。Hermes 额外做了一个上下文压缩功能，当对话快撑爆窗口时，自动总结早期内容，保留关键信息，让对话能持续更久。这一层没什么特别的。

**第二层：跨会话持久记忆**

关掉对话之后，重要信息不丢失。Agent 会把关键信息沉淀到持久化的文件里，一个记你的项目信息和技术偏好，一个记你这个人的画像。每次新对话启动时自动加载。

这一层 Claude 的 Memory 也在做，但 Hermes 多了一个设计细节：**主动沉淀机制**。它不等你说「帮我记住这个」，而是 Agent 自己周期性地评估当前对话里有没有值得永久记住的东西。

这个设计很聪明。你回想一下自己用 Claude 的场景，有多少次聊完了才想起来「刚才那个结论应该让它记住的」？用户的注意力是有限资源，Agent 的不是。

**第三层：历史会话全文检索**

第二层存的是提炼后的关键信息，高度压缩。但有时候你需要的不是摘要，而是原始对话。比如你说「我们上周讨论过的那个部署方案」，你需要 Agent 能翻到那次完整的对话记录。

Hermes 用 SQLite 的 FTS5 引擎对所有历史对话建了全文索引。当你引用过去的讨论时，Agent 搜索历史记录，找到相关片段，用 LLM 做摘要后注入当前上下文。

Claude 现在也有类似的能力，能搜索过去的对话。但差异在**数据主权**：Claude 的历史对话存在 Anthropic 的服务器上，Hermes 的存在你自己的机器上。对于重视数据隐私的开发者和企业来说，这个区别很关键。

**第四层：辩证式用户心智建模**

这一层最有意思，也是跟 Claude 差距最大的一层。

Claude 的 Memory 记的是什么？静态的事实标签。比如「用户是 AI 产品经理」「用户在做一个教育项目」。这些信息帮助 Agent 知道**你是谁**。

Hermes 通过集成一个叫 Honcho 的开源项目，试图建立一个动态的用户心智模型。它记的不是事实标签，而是**你的行为模式**。比如：你在面对模糊问题时倾向于先建框架再逐个击破，你对大而全但缺乏洞察的输出会不满意，你讨论技术问题时喜欢先看具体案例再听理论。

前者影响 Agent 给你什么内容，后者影响 Agent 用什么方式跟你协作。一个知道你是产品经理，一个理解你是怎么思考的。差别很大。

**那 Claude 为什么不做这个？**

不是做不到，是产品定位决定的。Claude 是云端服务，同时服务几百万用户，每个用户都做深度心智建模的计算成本太高了。而且存储这种深度画像会引发隐私争议。Hermes 是自托管的，一个实例只服务一个用户，成本可接受，数据也在你自己手里。

所以不是谁更先进的问题，是一对多的云服务适合做轻量标签，一对一的自托管适合做深度建模。场景不同，选择不同。

## Skills 系统：它怎么做到越用越聪明

Skill 就是一份告诉 Agent 遇到某类任务该怎么做的操作指南。比如「怎么做竞品分析」「怎么写一份调研报告」，把步骤、注意事项、常见坑都写清楚，Agent 加载后就能照着执行。

大多数 Agent 框架的 Skill 都是人写的。用户觉得某个任务值得复用，就手动编写一个 Skill 文档，告诉 Agent 遇到这类任务该怎么做。Claude 的 Skills 是这样，OpenClaw 也是这样。

Hermes Agent 做了一件不一样的事：**Skill 由 Agent 自己从经验中生成。**

你让它完成一个复杂任务，比如调研一个开源项目然后写分析报告。这个过程中它调用了七八个工具，中间踩了两个坑，最后找到了一条高效路径。任务完成后，Agent 自己回顾刚才的执行过程，把经验提炼成一个 Skill 文档存下来。

下次你再说「帮我调研一个项目」，它直接加载这个 Skill，不用从零推理。

更关键的是，这个 Skill 不是写完就放那了。每次被调用时，Agent 都会评估执行效果，如果发现了更好的方法，它会自动更新 Skill 文档。有用户报告说重复研究任务的速度在三个 Skill 积累后提升了 40%。

**这跟人工编写 Skills 有什么本质区别？**

三个层面。

**谁发起的。** 人工模式下，你得自己判断「这个值得做成 Skill」，自己决定怎么组织，自己动手封装。发起者是你，Agent 是执行者。Hermes 的模式下，Agent 自己判断、自己决定、自己生成。你甚至不需要知道这个过程发生了。

**谁来迭代。** 你手动写的 Skill，写完之后大概率就放在那了。说实话，你上次回去迭代一个已有的 Skill 是什么时候？大多数人都不会主动去更新。但 Hermes 的 Skill 在每次使用中都有可能被自动更新。迭代不需要你的注意力介入。

**知识来源不同。** 你写的 Skill 里放的是你的认知，你对这个任务的理解、你设计的流程、你的方法论。这些东西质量很高，但边界也是你的认知边界。Agent 自动生成的 Skill 沉淀的是执行层面的经验，哪个工具组合更高效、什么顺序成功率更高、容易踩什么坑。这些是你可能意识不到的细节。

**那人工写的 Skill 就没用了吗？**

恰恰相反。Agent 自动生成的 Skill 有一个致命短板：它学不到你的**价值判断**。

举个例子。如果你是一个做面试辅导的老师，你手动写的面试诊断 Skill 里会包含「什么是好的回答」的标准、你的教学方法论、你对不同类型候选人的判断策略。这些东西不是 Agent 能从执行经验中自动学到的。

所以最理想的模型不是二选一，而是分层结合：

| 
层级

 | 

谁来写

 | 

写什么

 |
| --- | --- | --- |
| 

方法论层

 | 

人工

 | 

价值判断、质量标准、领域知识

 |
| 

流程层

 | 

人工 + Agent

 | 

大框架人定，细节 Agent 优化

 |
| 

执行层

 | 

Agent 自动

 | 

工具选择、参数配置、错误处理

 |

人负责「什么是好的」，Agent 负责「怎么做更快」。人的注意力是有限资源，Agent 的不是。把不需要价值判断但需要持续迭代的部分交给 Agent，人专注在需要判断力的部分。

另外值得一提的是，Hermes 的 Skills 遵循一个叫 agentskills.io 的开放标准。Skills 不是私有格式，可以跨 Agent 共享，社区可以贡献和复用。这个思路跟 MCP 对工具层做的事情很像：标准化接口，降低迁移成本。

## Agent Loop 与终端后端：它怎么做到你不在它也在干活

先讲 Agent Loop，这是理解所有 Agent 产品的基础。

你跟 ChatGPT 或 Claude 对话时，背后发生的事情很简单：你发一条消息，它生成一段回复，结束。即使中间调了个搜索工具，本质上也是一条直线走下来的。

Agent Loop 不一样，它是一个循环。

你跟 Hermes Agent 说「帮我调研这个项目，写一份报告存到我的文件夹里」，Agent 内部会启动一个循环：先搜索 GitHub 页面拿到基本信息，判断信息不够，继续抓取 README 原文，判断还需要了解社区口碑，再去搜 Reddit 和 X 上的讨论，判断素材够了，开始写报告，写完存到指定目录，判断任务完成，停。

每一圈结束时，Agent 都在做一个决策：我该停了，还是该继续？下一步做什么？这个决策不是你告诉它的，是它自己判断的。

这就是 Agent Loop 的本质：**一个思考、行动、观察、再思考的自主循环，直到 Agent 自己认为任务完成。**

Claude 能不能做到？能。你让它调研一个项目，它也会搜索、抓取、再搜索、写报告，跑好几圈。

**区别不在能不能跑循环，而在循环的生命周期。**

Claude 的循环是被动触发、单次结束的。你发一条消息，它跑若干圈完成任务，给你回复，然后就停了。下一次行动必须等你再发一条消息。关掉对话，这次循环中积累的所有中间状态全部消失。

Hermes Agent 的循环可以自主触发、持久运行。三个关键差异：

**定时自主触发。** 你设一个 cron 任务告诉它「每天早上 8 点扫描 AI 领域的最新动态，写成简报发到我的 Telegram」。到了早上 8 点，没有人跟它说话，它自己启动循环，跑完流程，把结果推给你。Claude 做不到这个，没有人发消息，它就不会动。

**跨会话任务延续。** 一个任务太大，一次跑不完。Hermes Agent 可以今天跑一半，把中间状态存下来，明天接着跑。因为它有持久化的记忆和 Skill 系统。Claude 的每次对话本质上是独立的。

**循环前后有学习环节。** 前面讲过的 Skill 机制在这里串起来了。循环启动前，先检查有没有已有 Skill 能匹配当前任务，有的话直接加载。循环结束后，如果这次任务够复杂，Agent 会把执行过程沉淀成新的 Skill。标准 Agent Loop 是「思考、行动、观察」，Hermes 的是「加载 Skill → 思考、行动、观察 → 沉淀 Skill」，首尾两端是它独有的。

用一句话总结：Claude 是你按一下动一下的弹簧，Hermes 是你上好发条后自己持续转的齿轮。

**再讲终端后端，这个更实际。**

当 Agent 需要执行代码或跑命令时，它在哪里执行？

Claude Code 的答案很简单：在你的电脑上。你的终端就是它的终端。这意味着你电脑关了它就没法干活，而且它直接操作你的文件系统，有安全风险。

Hermes Agent 把执行环境抽象成了一个可插拔的层，提供六种选择：Local 本地执行，Docker 容器隔离，SSH 连到远程服务器，Daytona 和 Modal 是两个 Serverless 选项，最后还有面向科研超算的 Singularity。

其中 Daytona 和 Modal 的 Serverless 模式最值得关注。Agent 空闲时环境自动休眠，有任务来了秒级唤醒。你不需要 24 小时养一台服务器，只在 Agent 工作时才产生费用。社区里有人算过，轻度使用的月成本不到 1 美元。

这个设计背后的产品洞察很简单：**Agent 的电脑和你的电脑应该是两台不同的机器。** 交互界面和执行环境完全解耦。你在手机上通过 Telegram 发指令，Agent 在一台 VPS 上执行命令，两者互不依赖。

这也是从工具到基础设施的范式转变。工具是你打开才有用、关掉就停的。基础设施是 7×24 持续运行的。如果你在设计一个 Agent 产品，先想清楚一个问题：它在用户不在的时候应该做什么？如果答案是什么都不做，那它还是一个工具。

## 数据飞轮：它为什么免费

一个 AI 研究实验室，把自己最核心的 Agent 产品完全免费开源，MIT 协议，无遥测，无追踪，数据全在你本地。

它疯了吗？

拆完整个架构之后，我发现它比谁都精明。

**先说 Nous Research 是谁。**

在开源 AI 社区，Nous Research 是一个很有分量的名字。他们最出名的作品是 Hermes 系列微调模型，基于 LLaMA 等开源基座做指令微调，以推理能力强和指令遵循精准著称，是 Hugging Face 上下载量最高的开源微调模型之一。核心人物 Teknium 在开源 AI 社区的影响力很大。

所以他们做 Agent 不是跨界，是**顺着模型能力往应用层延伸**。

**再看它的商业逻辑。**

表面上看，Hermes Agent 是一个免费的开源产品。但如果你翻它的代码仓库，会发现两个很有意思的东西：一个叫 `trajectory_compressor.py` 的轨迹压缩器，一个叫 `tinker-atropos` 的 RL 训练环境子模块。

这两个东西是干什么的？把用户使用 Agent 过程中产生的工具调用轨迹，压缩成训练数据，用于强化学习训练更强的工具调用模型。

他们甚至没有试图隐藏这个意图。

整个商业飞轮长这样：

> ![](/assets/img/news/Agent【hermes-agent】，我从架构拆到商业模型-3.png)

1.  开源 Agent，免费给用户用。 吸引大量开发者安装和使用。
    
2.  用户使用 Agent 执行各种复杂任务。 每次任务都会产生工具调用轨迹数据。
    
3.  轨迹数据通过压缩和 RL 管线，训练更强的工具调用模型。
    
4.  更强的模型放到 Nous Portal，这是他们的付费推理平台，目前已经支持 400+ 模型。
    
5.  Agent 因为底层模型更强而变得更好用，口碑传播，吸引更多用户。
    

循环加速。

**Agent 是数据采集器，模型才是变现层。** 开源 Agent 是获客手段，Nous Portal 才是利润中心。

这个逻辑跟 Meta 开源 LLaMA 一脉相承：开源产品培育生态，生态产生的数据和反馈反哺核心业务。

**为什么这个飞轮能转起来？**

因为有一个关键前提：**工具调用轨迹数据是稀缺的。**

训练一个能熟练使用工具的 LLM，需要大量高质量的「模型使用工具完成任务」的示范数据。这种数据不像文本数据可以从互联网上爬取，它只能通过 Agent 实际执行任务来产生。用户越多，轨迹数据越多，数据质量和多样性越高，训练出的模型越强，Agent 表现越好，用户越多。

数据壁垒一旦建立起来，后来者很难追上。功能可以被抄，但飞轮跑起来之后积累的数据是抄不走的。

**竞争策略也值得一看。**

Hermes Agent 的代码仓库里有一个 `hermes claw migrate` 命令，一键迁移工具，能把你在 OpenClaw 里的人格文件、记忆、Skills、API Key、消息平台配置全部导入 Hermes。这是明牌在抢 OpenClaw 的用户。

而且这个策略确实在起作用。社区里大量「我从 OpenClaw 迁移过来了」的帖子，Reddit、X、YouTube 上到处都是对比评测。

**为什么用户愿意迁移？**

OpenClaw 先火是因为先发优势和更好的开箱体验。UI 更精致、文档更完善、上手门槛更低。如果你只是玩两天试试，OpenClaw 的第一印象确实更好。

但当用户从尝鲜进入长期使用阶段，差异就出来了。Hermes 的学习循环开始发挥作用，Skills 越积越多，记忆越来越深，Agent 明显变得更高效。而 OpenClaw 第 15 天和第 1 天比，没有本质变化。

**用久了之后，一个在进步，一个原地踏步。** 这就是用户迁移的核心原因。

短期体验 vs 长期价值，这是很多产品竞争中的经典张力。OpenClaw 赢在「好上手」，Hermes 赢在「用久了会变好」。

Nous Research 还有一个独立仓库 `hermes-agent-self-evolution`，用 DSPy + GEPA 自动优化 Agent 的 Skills 和 Prompts。不只是底层模型在进化，Agent 的行为策略本身也在进化。这意味着飞轮的加速不仅来自数据量增长，还来自 Agent 自身能力的提升。

产品+研究一体化的组织才能跑通这个飞轮。纯产品公司没有模型训练能力，纯研究机构没有用户量。Nous Research 两者兼具，这是它能设计出这个商业模型的前提。

* * *

## 这对我们意味着什么

拆完 Hermes Agent 的五层架构和商业模型，我觉得有三个判断值得分享。

**第一，Agent 赛道的竞争焦点正在从「能力多」转向「会进化」。**

2025 年的 Agent 产品比的是谁的工具多、谁能接更多平台、谁的 Prompt 写得更好。到了 2026 年，这些东西大家都有了，真正拉开差距的是：你的 Agent 用了一个月之后，比第一天强了多少？

Hermes Agent 给出了一个答案：Skills 自动生成 + 自我迭代 + 按需加载，构成一个完整的正反馈飞轮。每完成一个复杂任务，Agent 就比之前更强一点。这不是功能层面的创新，是产品范式的变化。

**第二，Agent 的产品形态正在从工具变成基础设施。**

传统的 AI 产品是你打开才有用、关掉就停的。Hermes Agent 的设计假设完全不同：Agent 应该有自己独立的运行环境，7×24 在线，用户不在它也在工作。

这个转变对产品设计的影响很大。如果你在做一个 Agent 产品，你需要回答一个问题：它在用户不在的时候应该做什么？如果答案是什么都不做，那你做的还是一个工具，不是一个 Agent。

**第三，开源不等于没有商业模式。**

Nous Research 的飞轮设计很精明：开源 Agent 采集轨迹数据，轨迹数据训练更强模型，更强模型放到付费平台变现。Agent 是入口，模型是利润。

这个模式给所有在 AI 领域创业的人一个启发：有时候免费送出去的产品，反而是你最有价值的资产。关键是你有没有能力把用户的使用行为转化成壁垒。

**最后说一句个人感受。**

我装 Hermes Agent 的时候第一次启动就报错了，折腾了一会儿才跑起来。跟 Claude 开箱即用的体验比，确实还有差距。但拆完它的架构之后，我理解了为什么那么多开发者愿意忍受这些粗糙，因为它提供了一个其他产品目前都没有的东西：**一个真的会变好的 Agent。**

至于它到底能不能兑现这个承诺，可能得用几个月时间才能验证。但至少方向是对的。
