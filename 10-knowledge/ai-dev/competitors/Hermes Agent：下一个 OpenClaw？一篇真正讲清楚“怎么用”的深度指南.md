在过去一年里，AI Agent 领域的形态逐渐清晰：  

从代码助手，到自动化工具，再到“长期运行的个人代理系统”。

短短两个月，GitHub ⭐突破 5 万。

👉 很少有人讲“第一天你该怎么用它”

# 一、Hermes Agent 到底是什么？（30 秒讲清）

  

  

一句话总结：

  

👉 Hermes = 一个可以长期运行、会自我进化的个人自动化代理

  

更具体一点：

  

*   部署在你自己的服务器 / 电脑上
    
*   通过 Telegram / Slack / CLI 和你交互
    
*   7×24 小时运行
    
*   自动执行任务、监控信息、做决策
    
*   最关键：它会“从经验中学习”
    

  

  

你可以把它理解为：

  

> 一个“会越用越聪明的自动化助理”，而不是一次性的 AI 工具

  

* * *

  

  

# 二、真正的核心：Learning Loop（学习闭环）

  

  

很多人以为 Hermes 的亮点是“自动化”。

  

其实不是。

  

真正的核心是：它会把经验变成能力。

  

它的机制非常简单但极其关键：

  

*   每执行大约 15 次工具调用
    
*   Hermes 会暂停
    
*   反思刚刚发生的事情：
    
      
    
    *   哪些步骤有效？
        
    *   哪些失败了？
        
    *   哪些可以优化？
        
    
      
    
*   然后生成一个 Skill（技能文件）
    

  

  

这些 Skill：

  

*   是 Markdown 文件（可读、可改）
    
*   存在本地：~/.hermes/skills/
    
*   本质是可执行流程
    

  

  

* * *

  

  

## 和传统“记忆”的区别

  

  

很多人用过类似 Claude Code 的记忆系统：

  

*   记录偏好（你喜欢 bullet points）
    
*   记住上下文
    

  

  

但 Hermes 不一样：

  

👉 它记住的是“怎么做事”

举个例子：

|   
系统

  
 |   

记住什么

  
 |
| --- | --- |
|   

普通 AI

  
 |   

你喜欢什么格式

  
 |
|   

Hermes

  
 |   

完整流程：搜索 → 筛选 → 总结 → 格式化

  
 |

这意味着：

  

👉 第 1 天：普通输出

👉 第 30 天：高度贴合你习惯的自动化流程

  

这是质变。

  

* * *

  

  

# 三、Hermes vs Claude Code vs OpenClaw

  

  

很多人最关心这个问题：到底选哪个？

  

先说结论：

  

> ❌ 不要选一个

> ✅ 正确做法：组合使用

  

* * *

  

  

## 1\. Claude Code

  

  

特点：

  

*   运行在代码仓库里
    
*   擅长写代码、测试、提交
    
*   开发能力极强
    

  

  

但：

  

*   不长期运行
    
*   不做日常自动化
    
*   不主动找你
    

  

  

👉 本质：开发代理

  

* * *

  

  

## 2\. OpenClaw

  

特点：

  

*   运行在服务器
    
*   支持消息、调度、工具调用
    
*   有“个人助手”的雏形
    

  

  

但：

  

*   ❌ 不会自我学习
    
*   ❌ 没有技能沉淀
    

  

  

👉 本质：自动化执行器

  

* * *

  

  

## 3\. Hermes Agent

  

  

特点：

  

*   有 OpenClaw 的所有能力
    
*   *   Learning Loop（学习闭环）
        
    
      
    

  

  

👉 本质：

  

> 一个“会进化”的自动化系统

  

* * *

  

  

## 一个关键观点（很重要）

  

  

👉 Claude Code + Hermes = 最优架构

  

*   Claude Code：负责写代码
    
*   Hermes：负责一切自动化
    

  

  

它们通过 MCP 协议共享工具：

  

👉 一套工具，两套大脑

# 四、7 个真实工作流（附可操作思路）

  

下面这部分，是这篇文章最有价值的地方。

  

* * *

  

  

## 1\. 每日智能简报（Morning Briefing）

  

你可以让 Hermes：

  

*   监控：
    
      
    
    *   邮件
        
    *   日历
        
    *   新闻 / 行业动态
        
    
      
    
*   每天早上自动推送总结到 Telegram
    

  

关键不是“简报”，而是：

  

👉 它会学习你真正关心什么

  

两周之后：

  

*   自动过滤无用邮件
    
*   提前标记重要会议
    
*   强化你常追问的话题
    

  

  

* * *

  

  

## 2\. 自动网页监控（替代人工巡检）

  

典型场景：

  

*   竞品价格
    
*   招聘信息
    
*   新闻变化
    
*   商品列表
    

  

  

Hermes 可以：

  

*   自动抓取网页
    
*   对比变化（diff）
    
*   只推送“新增或变化内容”
    

  

  

👉 你不再刷 10 个网站

👉 只看“变化”

  

* * *

  

## 3\. 一人公司（Single-Agent Company）

  

很多人尝试多 Agent：

  

*   市场 Agent
    
*   销售 Agent
    
*   工程 Agent
    

  

  

结果：

  

❌ 信息割裂

❌ 风格不统一

❌ 重复劳动

  

一个真实案例：

  

👉 最终只保留 一个 Hermes

  

它统一处理：

  

*   市场
    
*   外联
    
*   社区
    
*   日报
    

  

  

优势：

  

👉 所有上下文是共享的

  

* * *

  

  

## 4\. 自动知识库（Second Brain）

  

  

Hermes 内置类似 Andrej Karpathy 提出的知识管理思路：

  

结构是三层：

  

1.  原始数据（不修改）
    
2.  Wiki 页面（自动生成）
    
3.  结构规则（schema）
    

  

  

关键在于：

  

👉 Hermes 会持续维护这个知识库

  

*   新信息 → 自动更新旧页面
    
*   自动建立引用关系
    
*   标记冲突
    

  

  

👉 一个月后，你会得到一个“自进化知识系统”

  

* * *

  

  

## 5\. 自动实验系统（Auto Research）

  

  

这是最被低估的能力之一。

  

模式是：

  

*   改一点
    
*   测试结果
    
*   保留更优
    
*   循环
    

  

  

应用场景：

  

*   邮件打开率优化
    
*   落地页转化率
    
*   用户回复率
    

  

  

你只需要定义：

  

*   目标（KPI）
    
*   约束
    

  

  

👉 Hermes 自动迭代优化

  

* * *

  

  

## 6\. MCP 工具复用（零迁移）

  

  

Hermes 已支持 MCP：

  

👉 和 Claude Code 完全兼容

  

意味着：

  

*   你已有的：
    
      
    
    *   Google Workspace
        
    *   数据库
        
    *   API 工具
        
    
      
    

  

  

👉 全部直接复用

  

* * *

  

  

## 7\. 长期运行的自动任务系统

  

  

最简单的入门方式：

  

👉 只做一件事：

  

*   设置一个定时任务
    
*   让它每天运行
    

  

  

例如：

  

*   每天抓取行业 AI 新闻
    
*   每天总结 GitHub trending
    
*   每天整理客户邮件
    

  

  

关键：

  

👉 让它跑 2 周

  

因为：

  

> Hermes 的价值在“第 30 天”，不是第 1 天

  

* * *

  

  

# 五、90%的人第一步就错了：模型选择

  

  

这是最现实的问题。

  

很多人体验差，是因为：

  

👉 模型选错了

  

常见错误：

  

*   用小模型跑复杂任务
    
*   本地模型工具调用不稳定
    
*   幻觉严重
    

  

  

建议：

  

*   本地实验：Gemma 级别模型
    
*   真正使用：
    
      
    
    *   OpenAI
        
    *   OpenRouter
        
    *   Nous
        
    
      
    

  

  

👉 一句话：

  

> 模型决定下限，Learning Loop 决定上限

# 七、真正的使用建议（非常重要）

  

  

如果你只记住一件事，请记住这个：

  

* * *

  

  

## 不要一上来就搞复杂系统

  

  

正确方式：

  

1.  安装 Hermes
    
2.  接入 Telegram
    
3.  设置 一个任务（比如每日简报）
    
4.  运行 2 周
    

  

  

* * *

  

  

## 为什么？

  

  

因为 Hermes 的核心是：

  

👉 时间 +反馈 → 进化

  

不是 prompt engineering

不是一次性配置
