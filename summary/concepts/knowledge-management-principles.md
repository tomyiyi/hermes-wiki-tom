# 知识管理原则

## 核心理念
**知识要沉淀，要复用，要产出。存着不用等于没有。**

## 八条原则（来自实战经验）

1. **一个入口，多个终点**
   所有输入走 Notes/，统一流程分发到各目录

2. **结构够用就行**
   目录分到能找到东西就够了，不追求完美分类

3. **AI 建议，你拍板**
   分类反映你的思维结构，只有你最清楚一篇文章该放哪

4. **链接宜少不宜多**
   2-3 条强关联比 10 条弱关联有用得多

5. **对话也是知识**
   跟 AI 的深度讨论本身就在生产知识，别聊完就丢

6. **存着不用等于没有**
   知识库的终极标准：有没有帮你产出过东西？

7. **索引到目录，不到文件**
   目录结构稳定后几乎零维护

8. **不绑定工具**
   方法论通用，今天用 Claude，明天换别的也能跑

## RAG vs 知识编译

| | RAG | 知识编译（Wiki/Graphify） |
|--|-----|--------------------------|
| 查询方式 | 每次从零检索 | 先查已编译的知识 |
| 积累 | 无，每次独立 | 有，复利增长 |
| Token 消耗 | 高 | 低（Graphify 降至 1/72） |
| 认知持久性 | 无 | 有，跨会话 |

## 应用场景
- [[karpathy-llm-wiki]] — 文档知识管理
- [[graphify]] — 代码架构分析
- 个人知识库、工作流复盘、项目文档

## 行动层落地
知识最终要变成行动和产出：
- 投资笔记 → 交易决策
- 健康计划 → 每周训练
- 方法论文章 → 视频/课程
